    ## COMPONENT <br/> DESIGN 📂

    <FlexBox>
        #### 하나의 컴포넌트에서 <br/> 여러 뷰 컴포넌트를 관리하고 싶다.

        <FlexItemWrapper>
            SlideComp와 AboutComp는 다양한 내용을 담고 싶었다. 왜냐하면 컴포넌트가 많아지면 유지보수가 어렵다고 생각한다.

            <Highlight>SlideComp 컴포넌트</Highlight> 같은 경우 각 프로젝트마다 다른 내용을 가지지만, 레이아웃이 동일해야했고, 기술스택을 표현하는 아이콘도 겹치는 부분이 있다. 그래서 슬라이드를 구성하는데 필요한 카테고리로 컴포넌트를 생성했다 (title, insight, stack, repo). 슬라이드를 구성하는 컴포넌트에서 객체를 생성했고, 이 객체는 각기 다른 프로젝트 내용을 담고있는 컴포넌트들을 value 값으로 지정했다. key 값은 `src/data/project_data`에서 관리하는 id 값으로 지정해줬다.

            ##### 장점

            - 동일한 레이아웃을 사용하다 보니 style-components 코드를 많이 줄 일 수 있었다.
            - 하나의 컴포넌트에서 여러개의 컴포넌트를 관리한다는게 장점이자 단점이 될 수 있지만, 개인적으로 코드의 유지보수와 가독성이 향상시켰다고 생각한다. 왜냐하면 prop과 interface를 하나의 컴포넌트에서 관리할 수 있었다.
            - `StackComp`에서 동일한 아이콘을 재활용 할 수 있다.

            ##### 단점
            - 하나의 컴포넌트에서 관리하다보니 코드내용이 너무 길어졌다. 하지만 주석과 눈에 잘보이는 이모지로 섹션을 구분하려고 노력했다.

        </FlexItemWrapper>

    </FlexBox>

    <FlexBox>
        #### 컴포넌트 재사용

        <FlexItemWrapper>
            동일하게 사용되어야 하는 요소가 있다면 컴포넌트를 재사용하려고 노력했다. 그래서 이 컴포넌트를 쉽게 관리하고 싶어서 `common_comp`라는 디렉터리를 생성했고, `Index.tsx`컴포넌트에 공용으로 쓰이는 컴포넌트를 import해서 하나의 객체로 관리했다.
        </FlexItemWrapper>
